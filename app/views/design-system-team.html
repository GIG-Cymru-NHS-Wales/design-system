Design system team

The user-centred design (UCD) team at Digital Health and Care Wales maintains the design system. We research, develop and test designs based on evidence from users.

Contact the team by email for support, for example to make a request for a new design asset.

How we manage design asset requests

1. Receive the request

The UCD team receives a request for a new design asset. We add it to our backlog.

2. Evaluate

The UCD team meets weekly to:

  • review the backlog
  • decide which requests we can accept
  • prioritise accepted requests by how much value they will add and how complex they are

We then contact the requester to let them know if we're able to create the asset.

3. Design and test

Once a new design asset is agreed and prioritised, a team is assigned to:

  • research and design a working prototype
  • run usability and accessibility testing sessions with users
  • develop the design asset from the tested prototype

4. Publish

We publish finished design assets on the website with a description of how and when to use them.

We also make design asset documentation and version control available in the Design System repository on GitHub. Documentation for project work will be published on the DHCW SharePoint site (available only to DHCW staff).
